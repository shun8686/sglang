1问题场景

1.1 触发条件

在 NPU (Ascend) 平台上运行 Qwen3.5/Qwen3.6 等 GDN (Gated Delta Network) 混合线性注意力模型时，开启 chunked prefill 会导致输出精度严重下降。

NPU 硬件 + GDN 模型
(Qwen3.5/3.6/Next) + chunked prefill
(size=128) → 精度崩塌

1.2 业务流程

在 chunked prefill 场景下，长序列被切分为多个 chunk 逐块处理：

┌─────────────┐    SSM State₁    ┌─────────────┐    SSM State₂    ┌─────────────┐
│  Chunk 0    │ ───────────────→ │  Chunk 1    │ ───────────────→ │  Chunk 2    │
│  (128 tok)  │                  │  (128 tok)  │                  │  (128 tok)  │
└─────────────┘                  └─────────────┘                  └─────────────┘
      │                               │                               │
      └─── CoreAttnOut + last_recurrent_state ───┘

每个 chunk 之间需要正确传递 last_recurrent_state（SSM 循环状态张量），下一 chunk 读取该状态继续计算。若状态张量维度布局被意外修改，后续 chunk 读取时会得到错误的状态。

1.3 现象

场景chunked prefillceval 精度
修复前 size=128 精度崩溃（接近随机）
修复后 size=128 正常（与 size=-1 一致）

2根因分析

2.1 直接根因：多余的 transpose(-1, -2)

在 ascend_gdn_backend.py 的 forward_extend() 中，chunked prefill 每个 chunk 结束时都要将当前 SSM 状态写入缓存。旧代码对 speculative decoding 路径做了 transpose(-1, -2)：

修复前 — ascend_gdn_backend.py (旧代码已删除)
- if not forward_batch.spec_algorithm.is_none():
-     last_recurrent_state = last_recurrent_state.transpose(-1, -2).to(dtype)
- else:
-     last_recurrent_state = last_recurrent_state.to(dtype)
- ssm_states[cache_indices] = last_recurrent_state

修复后 — 统一处理，无 transpose
+ last_recurrent_state = last_recurrent_state.to(ssm_states.dtype, copy=False)
+ ssm_states[cache_indices] = last_recurrent_state

当 spec_algorithm 虽为默认值（NONE）但存在某些特定上下文（如某些 forward mode 下 spec_algorithm 对象被设置为非 NONE 状态），会错误触发 transpose，使状态下一次被读取时维度不匹配。

2.2 深层根因：NPU 的 temporal_state 维度约定与 GPU 不同

在 memory_pool.py 的 MambaPool.__init__() 中，speculative decoding 场景需要构造 intermediate_ssm_state_cache。NPU 上的 temporal_state 默认布局（HV, K, V 维度顺序）与 intermediate_ssm_state_cache 期望的布局不一致，需要额外做 transpose：

memory_pool.py — NPU 分支（新增）
+ if _is_npu:
+     temporal_state = temporal_state.transpose(-1, -2)
+     temporal_state_shape = (
+         *temporal_state_shape[:-2],
+         temporal_state_shape[-1],
+         temporal_state_shape[-2],
+     )
  # Cache intermediate SSM states per draft token during target verify
  intermediate_ssm_state_cache = torch.zeros(
      size=(num_mamba_layers, spec_state_size + 1,
            speculative_num_draft_tokens,
            temporal_state_shape[0],
            temporal_state_shape[1],
            temporal_state_shape[2]),
      ...)

2.3 因果链

NPU temporal_state 维度
与 GPU 顺序不同 → chunked prefill 下
transpose 操作冲突 → SSM state 布局
在 chunk 间不一致 → 下一 chunk 读取
错误状态 → 精度崩塌
