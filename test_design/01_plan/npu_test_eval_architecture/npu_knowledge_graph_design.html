1KG 解决 JSON 做不到的三件事

🔴 级联影响分析

JSON 能回答 "fuseep.py 属于哪些 feature？"（一层映射）。

KG 能回答: fuseep.py 的 bug 经由 DEPENDS_ON → TESTS_BY → SHARES_DEP 三层传播后，哪些关联特性的风险应同步上调？哪些测试可以提前预警？

🟠 盲点检测

JSON 能找到"风险评分最高"的特性。

KG 能发现: 介数中心性最高但风险评分最低的节点。比如 environ.py（评分 5/45）被 8+ 特性依赖——一个拼写错误就能瘫痪所有特性。JSON 永远看不到这个盲点。

🟡 多维模式匹配

JSON 的嵌套查询表达能力有限。

KG 能回答: 找出所有满足 "平台差异度>3 ∧ 静默错误 ∧ 零有效测试 ∧ 最近 30 天有变更" 的特性——这是四重条件的图模式匹配，SQL/JQ 表达会非常冗长。

📐 定位

KG 不是替代 JSON Baseline——它是在 JSON 之上加了一层关系推理引擎。JSON 做 90% 的日常查询（轻量快速，O(1) 属性访问），KG 做那 10% 的关键推理（跨层传播、盲点检测、模式发现）。

2.2 关系类型（7 类，各带权重）

关系名方向语义权重含义示例
IMPLEMENTS_IN Feature → SourceFile 特性在哪个文件中实现 该文件在特性中的代码量占比 (0-1) ascend_fuseep → fuseep.py (weight:0.8)
TESTED_BY Feature → TestCase 特性的测试 测试有效性评分 (0-5) ascend_fuseep → test_fuseep_oracle (weight:4)
DEPENDS_ON Feature → Feature 特性间依赖 调用深度/耦合度 (1-5) deepseek_v2 → ascend_fuseep (weight:4)
DEPENDS_ON Feature → Dependency 特性依赖外部组件 耦合强度 (1-5) ascend_fuseep → CANN 8.5.0 (weight:4)
AFFECTED_BY Feature ← Defect 缺陷影响了哪些特性 缺陷对该特性的影响程度 (1-5) ascend_fuseep ← BUG-2026-042 (weight:4)
FIXED_IN Defect → SourceFile 缺陷在哪个文件中被修复 修复涉及的文件数 (归一化) BUG-2026-042 → fuseep.py (weight:0.5)
COVERS TestCase → SourceFile 测试覆盖哪个源文件 覆盖的代码行占比 (0-1) test_fuseep_oracle → fuseep.py (weight:0.6)

4NetworkX 构建方案

4.1 从已有数据到图的一键构建

M0 全量分析已经产出 features.json、tests.json、risk_profiles.json、dependency_map.json。图和 JSON 是同一份数据的不同视图。

# .sglang-risk/scripts/build_graph.py

import json
import networkx as nx
from pathlib import Path
from hashlib import sha256

def build_risk_graph(baseline_dir: str) -> nx.MultiDiGraph:
    """从 JSON baseline 构建属性图。增量更新时只需重新运行此函数。"""
    G = nx.MultiDiGraph()
    baseline = Path(baseline_dir)

    # ── 加载基线数据 ──
    features   = json.loads((baseline / "features.json").read_text())
    tests      = json.loads((baseline / "tests.json").read_text())
    profiles   = json.loads((baseline / "risk_profiles.json").read_text())
    dep_map    = json.loads((baseline / "dependency_map.json").read_text())
    defects    = json.loads(
        (baseline.parent.parent / "db" / "defect_db.json").read_text()
    )

    # ── Step 1: 创建 Feature 节点 ──
    for f in features:
        p = profiles.get(f["name"], {})
        G.add_node(f["name"],
            type="Feature",
            category=f["category"],
            risk_score=p.get("composite_score", 0),
            probability=p.get("probability", {}).get("total", 0),
            impact=p.get("impact", {}).get("total", 0),
            test_depth=p.get("test_depth", "L3"),
            fingerprint=f.get("fingerprint", ""),
            n_tests=len(f.get("tests", [])),
            last_modified=f.get("last_modified", "")
        )

    # ── Step 2: 创建 SourceFile 节点 + IMPLEMENTS_IN 边 ──
    for path, info in dep_map.items():
        if info.get("is_test"):
            continue
        file_id = path.split("/")[-1]
        G.add_node(file_id,
            type="SourceFile",
            path=path,
            complexity=0,  # 后续由 Agent 填充
            change_freq_90d=0,
            n_bugs=0,
            file_type=info.get("file_type", "source")
        )
        # IMPLEMENTS_IN: Feature → SourceFile
        for feat_name in info.get("features", []):
            if G.has_node(feat_name):
                G.add_edge(feat_name, file_id,
                    type="IMPLEMENTS_IN",
                    weight=1.0 / len(info["features"])  # 按特性数均分
                )

    # ── Step 3: 创建 TestCase 节点 + TESTED_BY / COVERS 边 ──
    for t in tests:
        test_id = t["test_file"].split("/")[-1].replace(".py", "")
        G.add_node(test_id,
            type="TestCase",
            test_type=t.get("test_type", "unknown"),
            quality_score=t.get("quality_score", 0),
            assertion_type=t.get("assertion_type", "threshold"),
            has_oracle=t.get("has_reference_oracle", False),
            ci_suite=t.get("ci_suite", "")
        )
        # TESTED_BY: Feature → TestCase
        for feat_name in t.get("features_tested", []):
            if G.has_node(feat_name):
                G.add_edge(feat_name, test_id,
                    type="TESTED_BY",
                    weight=t.get("quality_score", 1)
                )
        # COVERS: TestCase → SourceFile (从 dep_map 反查)
        for path, info in dep_map.items():
            if info.get("is_test") and path == t["test_file"]:
                for covered_file in info.get("covers_files", []):
                    file_id = covered_file.split("/")[-1]
                    if G.has_node(file_id):
                        G.add_edge(test_id, file_id,
                            type="COVERS",
                            weight=info.get("coverage_ratio", 0.5)
                        )

    # ── Step 4: 创建 Defect 节点 + AFFECTED_BY / FIXED_IN 边 ──
    for d in defects:
        G.add_node(d["bug_id"],
            type="Defect",
            severity=d.get("severity", 3),
            failure_mode=d.get("failure_mode", "unknown"),
            is_silent=d.get("is_silent", False),
            date_fixed=d.get("date_fixed", ""),
            commit_fixed=d.get("commit_fixed", "")
        )
        for feat_name in d.get("features_affected", []):
            if G.has_node(feat_name):
                G.add_edge(d["bug_id"], feat_name,
                    type="AFFECTED_BY",
                    weight=d.get("severity", 3)
                )
        for file_path in d.get("files_fixed", []):
            file_id = file_path.split("/")[-1]
            if G.has_node(file_id):
                G.add_edge(d["bug_id"], file_id,
                    type="FIXED_IN",
                    weight=1.0 / len(d["files_fixed"])
                )

    # ── Step 5: 推断 DEPENDS_ON (Feature → Feature) ──
    # 两个 Feature 共享 SourceFile → 它们有间接耦合
    file_to_features = {}
    for u, v, data in G.edges(data=True):
        if data.get("type") == "IMPLEMENTS_IN":
            file_to_features.setdefault(v, []).append(u)

    for file_id, feat_list in file_to_features.items():
        if len(feat_list) > 1:
            for i, a in enumerate(feat_list):
                for b in feat_list[i+1:]:
                    if not G.has_edge(a, b):
                        G.add_edge(a, b, type="DEPENDS_ON", weight=2,
                            via=f"shared_file:{file_id}")
                        G.add_edge(b, a, type="DEPENDS_ON", weight=2,
                            via=f"shared_file:{file_id}")

    # ── Step 6: 创建 Dependency 外部节点 ──
    deps = {
        "CANN":       {"version":"8.5.0","dep_type":"runtime","upgrade_risk":4},
        "torch_npu":  {"version":"2.5.1","dep_type":"python","upgrade_risk":3},
        "sgl_kernel_npu":{"version":"2026.05","dep_type":"kernel","upgrade_risk":4},
        "HCCL":       {"version":"8.5.0","dep_type":"comm","upgrade_risk":3},
    }
    for name, props in deps.items():
        G.add_node(name, type="Dependency", **props)
        # 所有 NPU 特性都依赖 CANN 和 torch_npu
        for feat_name in [f["name"] for f in features]:
            if G.has_node(feat_name):
                coupling = 5 if name in ("CANN", "torch_npu") else 3
                G.add_edge(feat_name, name, type="DEPENDS_ON", weight=coupling)

    # ── Step 7: 计算图度量 ──
    for node in G.nodes():
        G.nodes[node]["degree"] = G.degree(node)
    for node in G.nodes():
        # 介数中心性（用无向图近似）
        G_undirected = nx.Graph()
        for u, v in G.edges():
            G_undirected.add_edge(u, v)
        if G_undirected.number_of_nodes() > 0:
            bc = nx.betweenness_centrality(G_undirected)
            G.nodes[node]["betweenness"] = bc.get(node, 0)

    return G


# ── 使用 ──
if __name__ == "__main__":
    G = build_risk_graph(".sglang-risk/baselines/latest/")
    print(f"Nodes: {G.number_of_nodes()}, Edges: {G.number_of_edges()}")

    # 导出为 JSON（供可视化 / 前端消费）
    data = nx.node_link_data(G)
    Path(".sglang-risk/graph/latest_graph.json").write_text(

    )

4.2 增量更新图

图的增量更新比 JSON 更简单——因为图本身就是集合运算的自然表达。

def update_graph_delta(G: nx.MultiDiGraph, delta: dict) -> nx.MultiDiGraph:
    """对已有图应用 delta，返回更新后的图。"""

    for change in delta["changes"]:
        # 新增节点
        if change["type"] == "new_feature":
            G.add_node(change["name"], type="Feature", **change["props"])

        # 更新节点属性
        elif change["type"] == "risk_score_change":
            G.nodes[change["feature"]]["risk_score"] = change["new_value"]

        # 新增边
        elif change["type"] == "new_test":
            G.add_node(change["test_id"], type="TestCase", **change["props"])
            G.add_edge(change["feature"], change["test_id"],
                type="TESTED_BY", weight=change.get("quality_score", 1))

        # 移除边（测试被删除）
        elif change["type"] == "test_removed":
            if G.has_edge(change["feature"], change["test_id"]):
                G.remove_edge(change["feature"], change["test_id"])

        # 新增缺陷
        elif change["type"] == "new_defect_recorded":
            G.add_node(change["bug_id"], type="Defect", **change["props"])
            for feat in change["features_affected"]:
                G.add_edge(change["bug_id"], feat,
                    type="AFFECTED_BY", weight=change["severity"])

    # 重算图度量（轻量，~80 节点）
    G_undirected = nx.Graph()
    for u, v in G.edges(): G_undirected.add_edge(u, v)
    bc = nx.betweenness_centrality(G_undirected)
    for node in G.nodes():
        G.nodes[node]["betweenness"] = bc.get(node, 0)

    return G

Q1 单文件变更影响面

def impacted_features(G, file_id):
    # 反向追溯: file ← IMPLEMENTS_IN ← Feature
    direct = {u for u,v,d in G.in_edges(file_id) if d['type']=='IMPLEMENTS_IN'}
    # 正向传播: Feature → DEPENDS_ON → Feature (2 跳)
    all_impacted = set(direct)
    for f in direct:
        for _, neighbor, d in G.out_edges(f, data=True):
            if d['type'] == 'DEPENDS_ON':
                all_impacted.add(neighbor)
    return all_impacted

Q2 依赖升级爆炸半径
"CANN 升级到 8.6.0，影响哪些特性？按风险排序" 
def cann_upgrade_blast_radius(G, dep_name):
    features = []
    for _, feat, d in G.in_edges(dep_name):
        if d['type'] == 'DEPENDS_ON':
            features.append((
                feat,
                G.nodes[feat]['risk_score'],
                d['weight']  # coupling strength
            ))
    return sorted(features, key=lambda x: x[1], reverse=True)

Q3 高介数低风险盲点

def blind_spot_detection(G, bc_threshold=0.6, risk_threshold=20):
    blinds = []
    for node, data in G.nodes(data=True):
        bc = data.get('betweenness', 0)
        risk = data.get('risk_score', 0)
        if bc > bc_threshold and risk < risk_threshold:
            blinds.append((node, data['type'], bc, risk))
    return sorted(blinds, key=lambda x: x[2], reverse=True)

def high_coupling_no_test(G):
    for node, data in G.nodes(data=True):
        if data['type'] != 'SourceFile': continue
        out_degree = G.out_degree(node)  # 影响多少个 feature
        # 检查是否有 COVERS 边
        covered = any(
            d['type'] == 'COVERS'
            for _, _, d in G.in_edges(node, data=True)
        )
        if out_degree >= 3 and not covered:
            yield node, out_degree, data['path']

类型三：多维模式匹配

Q5 "完美风暴"检测
平台差异度>3 ∧ 静默错误模式 ∧ 零有效测试 ∧ 近30天变更 
def perfect_storm(G, days=30):
    from datetime import datetime, timedelta
    cutoff = (datetime.now() - timedelta(days=days)).isoformat()
    results = []
    for node, data in G.nodes(data=True):
        if data['type'] != 'Feature': continue
        # 条件1: 平台差异度 > 3
        has_divergence = any(
            d['weight'] > 3 and G.nodes[v].get('platform_divergence') == 'fully_npu_private'
            for _, v, d in G.out_edges(node, data=True) if d['type'] == 'IMPLEMENTS_IN'
        )
        # 条件2: 历史有静默错误缺陷
        has_silent_defect = any(
            G.nodes[u]['is_silent']
            for u, _, d in G.in_edges(node, data=True) if d['type'] == 'AFFECTED_BY'
        )
        # 条件3: 零有效测试
        n_effective = data.get('n_effective_tests', 0)
        # 条件4: 30天内有变更
        recent = data.get('last_modified', '') > cutoff
        if has_divergence and has_silent_defect and n_effective == 0 and recent:
            results.append((node, data['risk_score']))
    return sorted(results, key=lambda x: x[1], reverse=True)

def defect_hotspots(G):
    file_defects = {}
    for node, data in G.nodes(data=True):
        if data['type'] != 'Defect': continue
        for _, file_id, d in G.out_edges(node):
            if d['type'] == 'FIXED_IN':
                file_defects.setdefault(file_id, []).append({
                    'bug_id': node,
                    'failure_mode': data['failure_mode'],
                    'is_silent': data['is_silent']

    # 按缺陷数排序
    return sorted(file_defects.items(), key=lambda x: len(x[1]), reverse=True)

类型四：测试策略优化

Q7 测试有效性缺口排序
"按 (风险 × (5 - 测试有效性)) 排序，优先补哪个特性的测试？" 
def test_priority(G):
    priority = []
    for node, data in G.nodes(data=True):
        if data['type'] != 'Feature': continue
        test_edges = [
            d['weight'] for _, _, d in G.out_edges(node, data=True)
            if d['type'] == 'TESTED_BY'
        ]
        max_effectiveness = max(test_edges) if test_edges else 0
        gap = 5 - max_effectiveness
        score = data['risk_score'] * gap
        priority.append((node, data['risk_score'], max_effectiveness, gap, score))
    return sorted(priority, key=lambda x: x[4], reverse=True)

类型五：变更影响传播

Q8 n-hop 影响传播
"从这个文件出发，3 跳内的所有受影响节点" 
def n_hop_impact(G, start_file, hops=3):
    # 找到源文件节点
    file_node = next(n for n,d in G.nodes(data=True)
                     if d.get('path','').endswith(start_file))
    # BFS 传播（沿 IMPLEMENTS_IN, DEPENDS_ON, AFFECTED_BY）
    visited = set()
    frontier = {file_node}
    for _ in range(hops):
        next_frontier = set()
        for node in frontier:
            for _, neighbor, d in G.out_edges(node):
                if d['type'] in ('IMPLEMENTS_IN','DEPENDS_ON','AFFECTED_BY'):
                    if neighbor not in visited:
                        next_frontier.add(neighbor)
        visited.update(frontier)
        frontier = next_frontier - visited
    return visited | frontier

6KG 与增量框架的融合

6.1 变更影响传播：从单层到多层

当前的 dependency_map.json 只做一层映射 (file → feature)。KG 做多层传播，二者串联而不是替代。

fuseep.py Step 2 JSON → {ascend_fuseep} (1个) Step 3 KG propagation 1-hop → {deepseek_v2, qwen3_moe, glm4_moe} (DEPENDS_ON, +3个) Step 3 KG propagation 2-hop (共享缺陷模式) → {dynamic_quant} (SHARES_DEFECT_PATTERN, +1个) Step 4 去重 → {ascend_fuseep, deepseek_v2, qwen3_moe, glm4_moe, dynamic_quant} 共5个 Feature = 5个 Risk-Scorer Agent

6.2 增量循环中的角色分工

✅ 关键效果

KG 的传播分析让受影响特性从 1 个提升到 5 个（上例），同时不增加 Agent 的无效调用（指纹比对过滤掉了真正没变的那些）。这实现了影响面全覆盖 + 计算成本严格正比于真实变更量。

11总结

📐 设计决策

  • 属性图模型（5节点 + 7关系），不引入 RDF/OWL
  • NetworkX 优先，~80 节点内存图足够
  • KG 是 JSON 的补充层，不替代
  • 图从已有 baseline JSON 自动构建，零额外人工

🎯 核心价值

  • 级联影响: 1-hop JSON + n-hop KG → 无遗漏
  • 盲点检测: 高介数低风险节点（JSON 看不到）
  • 模式匹配: "完美风暴"等多条件组合查询
  • 图度量: 介数中心性 / PageRank / 社区发现

🔄 与增量框架协同

  • git diff → JSON 直接映射 (O(1))
  • JSON 结果 → KG n-hop 传播 (O(E))
  • 合并去重 → 指纹比对 → Agent 增量评分
  • 评分结果 → 更新图 (O(Δ))

✅ 四份文档分工明确

Doc 1: 覆盖审计 → 知道"测没测"
Doc 2: 风险策略 → 知道"哪里多测、哪里少测、怎么测"
Doc 3: 增量架构 → 知道"变化后如何高效更新"
Doc 4: 知识图谱 → 知道"隐藏的关系、级联的风险、结构性的盲点"
Doc 3 和 Doc 4 互为补充——增量架构负责效率（只算变化的），KG 负责深度（发现隐含的）。
